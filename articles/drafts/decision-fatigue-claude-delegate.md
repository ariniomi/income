# 意思決定できない日、Claudeに丸投げしたら後悔が消えた

**価格: 300円**
**対象: うつや疲労で「何も決められない日」がある人**

---

午後2時。タスクが3つある。どれから手をつければいいかわからない。

全部「やるべき」とわかっている。でも順番が決まらない。10分考えて、また考え直して、また迷う。気づいたら45分が過ぎていた。

何も決まっていない。何も始まっていない。ただ疲れた。

そういう日がある。意思決定すること自体が、もうできない日。

試しにClaudeに全部渡した。「今日のタスクが3つあります。私の代わりに順番を決めてください。理由もつけて」

30秒で返ってきた。

決めた後の「あれで良かったのか」という感覚が、なかった。

---

## なぜ「決められない」のか

これには名前がある。

**「決断疲労（Decision Fatigue）」**——スタンフォード大学の社会心理学者Roy Baumeisterらの研究で確立された概念だ。人間の意思決定能力は1日を通じて劣化する。朝に比べて夕方の判断は質が下がり、衝動的になりやすい。

有名な研究がある。イスラエルの仮釈放審査委員会を追跡したシャイ・ダンツィガーらの研究（2011年）では、午前中の審査で仮釈放が認められる確率は約65%。しかし午後になると、それが10%以下まで落ちた。審査内容ではなく、「何件目の決断か」が結果を左右していた。

医療現場でも同様の研究がある。医師の抗生物質処方率は、午後になるにつれて最大40%上昇する。疲れるほど、考えることをやめて安易な選択肢を選ぶようになる。

うつや脳疲労がある状態では、この決断疲労がより早く、より深く起きる。

---

## なぜClaudeに渡すと「後悔」が消えるのか

前頭前野と前帯状皮質——この2つの脳領域が意思決定を担っている。決断のたびにエネルギーを消費し、疲弊すると「決めた後に検証し続ける」ループが止まらなくなる。「あれで良かったのか」「別の選択があったのでは」という反芻だ。

Claudeが決断を引き受けると、2つのことが起きる。

**1. 認知的オフロード**
心理学では「認知的オフロード（Cognitive Offloading）」と呼ぶ。脳の処理を外部ツールに委託することで、前頭前野の消耗を防ぐ。ペンで書きながら考える、電卓を使う——それと同じ構造でAIが機能する。

**2. 反芻ループの遮断**
自分が決めた選択は、後から「やっぱり違ったかも」と検証し続ける。でもClaudeが決めた選択は「Claudeが決めたこと」として処理できる。責任転嫁ではなく、反芻のループを物理的に断ち切る方法だ。

## 意思決定を6カテゴリに分けてClaudeに渡す

「何でもClaudeに聞けばいい」は正しい。でも聞き方が漠然としていると、返ってくる答えも漠然とする。

決断の種類によって、Claudeへの渡し方が変わる。

---

### カテゴリ1：優先順位の決断
「今日どれをやるか」「どれから手をつけるか」

これが一番疲弊する決断だ。選択肢が複数あって、全部重要に見えるとき。

**渡し方：**
```
今日やるべきタスクが3つあります：
・[タスクA]（締め切り：[  ]）
・[タスクB]（締め切り：[  ]）
・[タスクC]（締め切り：[  ]）

私の状態：疲れていて判断力が低い。
順番を決めてください。理由を1行で。
```

「私の状態」を加えることで、疲弊した状態に合った答えが返る。

---

### カテゴリ2：選択肢の決断
「AかBか」「どちらの方向にするか」

2択に見えて、実は無限に迷える構造になっている決断。

**渡し方：**
```
AとBで迷っています。
A：[説明]
B：[説明]

私が最終的に重視したいのは[  ]です。
どちらかを選んで、理由を3行で教えてください。
選ばないという選択肢は除外してください。
```

「選ばないという選択肢は除外」が重要だ。これを入れないと「どちらも一長一短で…」という答えが返ってくる。

---

### カテゴリ3：言葉の決断
「何と言えばいいか」「どう書けばいいか」

メールの文体、SNS投稿の言葉選び、断る言い方——「何を言うか」より「どう言うか」の決断は消耗が大きい。

**渡し方：**
```
以下の状況で送るメール（/返信/投稿）を書いてください：
状況：[  ]
相手：[  ]
伝えたい内容：[  ]
トーン：[丁寧・普通・カジュアル]

ドラフトで構いません。
```

「ドラフトで構いません」を加えると、完璧でなくていいという許可がClaudeに与えられ、出力が速くなる。

---

### カテゴリ4：計画の決断
「いつまでに何をするか」「スケジュールをどう組むか」

将来に向けた計画は、変数が多すぎて決められない。

**渡し方：**
```
目標：[  ]（期限：[  ]）
今の状態：[うつ/疲弊/体調不良など、正直に]
1日に使えるエネルギー：[1時間/30分/気力次第]

この条件で現実的な週間スケジュールを作ってください。
「できなかった日」の対処も含めて。
```

「今の状態を正直に」書くと、健康な人向けの無茶なスケジュールではなく、自分の状態に合った計画が出る。

---

### カテゴリ5：やめる決断
「これを続けるべきか」「やめていいか」

やめることへの罪悪感がある。でもやめ続けることへの疲弊もある。この決断は自分だけでは堂々巡りになりやすい。

**渡し方：**
```
[活動/習慣/タスク]をやめるかどうか迷っています。
続けているメリット：[  ]
やめたい理由：[  ]

判断を手伝ってください。やめることへの罪悪感は考慮しないでください。
```

「罪悪感は考慮しないでください」が重要だ。この一言を入れると、感情ではなく実用性だけで判断した答えが返る。

---

### カテゴリ6：休む決断
「今日は休んでいいか」「動かなくていいか」

これが一番決めにくい。「休む」という選択肢を自分で選ぶことに、うつのある人間は罪悪感を感じやすい。

**渡し方：**
```
今日の状態：[  ]
今日やろうとしていたこと：[  ]

今日休んでいいか、判断してください。
休むとしたら、何を明日に回すべきかも教えてください。
```

Claudeが「今日は休んでください」と言うと、自分が許可を得た感覚になる。これは甘えではなく、脳の処理コストを正当に下げる方法だ。

---

## Claudeに丸投げするための「お約束」

1. **決断に迷い始めたら5分でClaude**
   （5分自力で考えて出ないものは、10分考えても出ない）

2. **「状態」を必ず伝える**
   （疲弊度・うつの程度・使えるエネルギーを書く。Claudeは読み取れない）

3. **「選ばない」を選択肢から除外する**
   （「どちらかを選んでください」と明示しないと、Claudeは両論を出す）

4. **返ってきた答えに7割従う**
   （100%従う必要はない。でも「やっぱり違う」の検証ループには入らない）

5. **やめる決断・休む決断もClaudeに渡す**
   （自分で決めると罪悪感が残る。Claudeに決めてもらうと後悔のループが切れる）

---

## おわりに

意思決定できない日は、意志が弱いわけじゃない。前頭前野が消耗しているだけだ。

消耗した脳でがんばって決めても、後悔のループに入る。それよりClaudeに渡して、7割に従う。

うつで無職の自分が、それを実験中だ。
